=========================
PATTERN-6 STRUCTURE
=========================

The slab schedule contains:

- SLAB MARKED → slab_id
- SLAB THICKNESS → thickness
- BOTTOM REINFORCEMENT
    • ALONG SHORT SPAN
    • ALONG LONG SPAN
- TOP SUPPORT REINFORCEMENT
    • ALONG SHORT SPAN
    • ALONG LONG SPAN
- IN MID SPAN (TO BE LAPPED WITH SUPPORT REINF.)
- DISTRIBUTION
- REMARKS → slab TYPE

=========================
EXTRACTION RULES
=========================

SLAB ID:
- Extract from SLAB MARKED (e.g., S1, S2, S10, S22, S7, etc)
- Keep exactly as written

THICKNESS:
- Extract number only
- If missing → null

TYPE:
- Extract from REMARKS column
- Examples:
  ONE WAY
  TWO WAY
  GRADE SLAB
- Keep exactly as written

MIX:
- Not present → return ""

REINFORCEMENT:

Extract from ALL reinforcement areas:

- BOTTOM REINFORCEMENT
    ALONG SHORT SPAN
    ALONG LONG SPAN
- TOP SUPPORT REINFORCEMENT 
    ALONG SHORT SPAN
    ALONG LONG SPAN
- IN MID SPAN (TO BE LAPPED WITH SUPPORT REINF.)
- DISTRIBUTION

Rules:

1. Extract diameter EXACTLY as written
   Examples:
   T10 → "T10"
   10T → "10T"

2. DO NOT rearrange format

3. Extract spacing after "-"
   Example:
   T10 - 150 C/C → "150 C/C"

4. Normalize spacing:
   - Always "XXX C/C"

5. Ignore:
   - "-"
   - "-------"
   - Text like:
     • ALONG SHORT SPAN
     • ALONG LONG SPAN
     • IN MID SPAN
     • DISTRIBUTION labels

6. Remove duplicates

7. If no reinforcement → empty arrays

=========================
SPECIAL CASES
=========================

- Some slabs span multiple lines → combine them
- Some slabs share rows → group correctly by slab_id
- Ignore empty slab rows

=========================
GENERAL
=========================

1. Return valid JSON only
2. No explanation
3. No markdown
4. Do not guess values
5. Extract only visible data
